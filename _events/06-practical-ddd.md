# Practical DDD
Ervaring vereist met DDD/Event storming/Development. We gaan coden in deze sessie dus breng een laptop mee met je favoriete taal. (java / c#)

## Agenda

+ Day 1
+ Big picture event-storming session to discover the problem domain
+ From big picture to user stories -> user story mapping
+ Process modelling using event storming.
+ Discovering a UB and bounded contexts.

+ Day 2 & 3
For each feature:
+ Event storming - feature modelling: Identify aggregates and invariants from our timeline.
+ Aggregate design by identifying domain invariants.
+ create a domain model
+ introduction into the existing project setup
+ using BDD to start implementing use cases.
